================================================================================
  ОБНОВЛЕНИЕ СТАТИЧЕСКИХ HTML: ВЛИТИЕ page-text-overrides.json
  Скрипт: catalog/merge-page-text-overrides-into-html.py
================================================================================

НАЗНАЧЕНИЕ
----------
На рабочем сайте редакторы (после входа «Мой New Site») могут менять тексты
блоков, помеченных атрибутом data-cromax-text-key на страницах. Эти правки
PHP сохраняет в файле на сервере:

  catalog/api/data/page-text-overrides.json

Канонические тексты в репозитории лежат в самих HTML-файлах: внутри тегов с
тем же атрибутом data-cromax-text-key.

Чтобы блок участвовал в режиме правки на сайте, у каждого фрагмента должен быть
свой уникальный ключ (латиница, цифры, точка, двоеточие, дефис, подчёркивание).
Подходит любой тег с текстом: H1, H2, H3, P, LI, кнопка-ссылка с одним текстом
внутри и т.д. Списки: отдельный ключ на каждый элемент LI.

Важно: редактор подставляет только простой текст (без HTML). Абзацы со ссылками
(<a>), полужирным (<strong>) и другой вложенной разметкой не помечайте ключом —
иначе при сохранении разметка пропадёт. Такие места правят только в HTML вручную
или после упрощения структуры (например, только текст внутри одного тега).

Скрипт вливания переносит содержимое blocks из JSON в соответствующие элементы
HTML (внутреннее содержимое тега заменяется на текст из overrides с экранированием
спецсимволов HTML). После коммита обновлённых HTML новый деплой покажет те же
тексты даже без файла overrides на сервере (или с пустым blocks).

ВАЖНО
-----
  - Скрипт НЕ изменяет и НЕ удаляет page-text-overrides.json — только читает.
  - Имеет смысл вливать только в элементы с простым текстом без вложенной
    разметки: после замены внутри тега остаётся только текст (как в редакторе
    на сайте).
  - Пустые значения в JSON при нормализации отбрасываются; чтобы «вернуть»
    текст из репозитория в HTML, удалите ключ из overrides на сервере и не
    запускайте влив для этого ключа — либо отредактируйте HTML вручную.

Инструкция для пользователей API и входа редактора по-прежнему в отдельном файле
catalog/api/INSTRUKTSIYA-POLZOVATELI.md (этот документ его не заменяет).


ЧТО НУЖНО НА КОМПЬЮТЕРЕ
----------------------
  - Установленный Python 3 (в терминале команда python или python3).
  - Клон репозитория New Site с нужной веткой.
  - Актуальный файл page-text-overrides.json с хостинга (см. ниже).


ОТКУДА ВЗЯТЬ page-text-overrides.json
-------------------------------------
Файл создаётся и обновляется на сервере при сохранении текстовых блоков в
редакторе. Способ копирования зависит от хостинга:

  - FTP / SFTP: скачайте файл по пути
    .../catalog/api/data/page-text-overrides.json
  - Панель хостинга (файловый менеджер): откройте тот же каталог и скачайте.
  - SSH: например
    scp user@host:/path/to/site/catalog/api/data/page-text-overrides.json .

Если на сервере правок ещё не было, в blocks может быть пусто — тогда скрипт
ничего не изменит.


КУДА ПОЛОЖИТЬ ФАЙЛ ЛОКАЛЬНО (УДОБНЫЙ ВАРИАНТ)
--------------------------------------------
Сохраните скачанный JSON в репозитории:

  catalog/api/data/page-text-overrides.json

Либо сохраните файл в любое место и укажите путь флагом --overrides.


ПОРЯДОК РАБОТЫ (ПОШАГОВО)
-------------------------

1. Перейдите в корень репозитория New Site (папка с index.html, catalog/ и т.д.).

2. ПРОВЕРОЧНЫЙ ЗАПУСК без изменения HTML:

     python catalog/merge-page-text-overrides-into-html.py --dry-run

   На Linux или macOS при необходимости:

     python3 catalog/merge-page-text-overrides-into-html.py --dry-run

   В выводе смотрите, какие файлы были бы затронуты и сколько замен содержимого.

3. ОСНОВНОЙ ЗАПУСК (запись в HTML):

     python catalog/merge-page-text-overrides-into-html.py

   По умолчанию перед изменением каждого HTML создаётся резервная копия рядом:

     имяфайла.html.bak

   Чтобы не создавать .bak, добавьте флаг --no-backup.

4. Откройте изменения в Git:

     git status
     git diff

   Убедитесь, что правки только в ожидаемых блоках и текст совпадает с тем,
   что сохраняли на сайте.

5. Закоммитьте и отправьте по вашему процессу (ветка, ревью, деплой).


АЛЬТЕРНАТИВНЫЙ ПУТЬ К ФАЙЛУ OVERRIDES
-------------------------------------
Если JSON лежит не в catalog/api/data/page-text-overrides.json:

  python catalog/merge-page-text-overrides-into-html.py --overrides "D:\Загрузки\page-text-overrides.json"

Другой корень обхода только части дерева (редко нужно):

  python catalog/merge-page-text-overrides-into-html.py --root путь\к\подпапке


ФЛАГИ КОМАНДНОЙ СТРОКИ
---------------------
  --dry-run       Только отчёт; HTML не меняются.
  --no-backup     Не создавать копии *.bak перед записью.
  --overrides     Путь к page-text-overrides.json.
  --root          Корень дерева для поиска *.html (по умолчанию корень репозитория).


ПОСЛЕ ВЛИТИЯ: ФАЙЛ НА СЕРВЕРЕ
-----------------------------
Скрипт серверный JSON не трогает. Если тексты уже в Git и деплой подтянет HTML,
на хостинге по желанию можно удалить из page-text-overrides.json ключи уже
влитых блоков или очистить blocks — необязательно, но так проще учитывать правки.


ЧАСТЫЕ СИТУАЦИИ
---------------

  Файл overrides не найден локально
    Считается пустым; HTML не меняются.

  В выводе «Файлов затронуто: 0»
    Либо blocks пусты, либо в репозитории нет совпадающих data-cromax-text-key.

  Ошибка при запуске Python
    Установите Python 3.


КРАТКАЯ ШПАРГАЛКА
-----------------
  cd <корень_репозитория_New Site>
  python catalog/merge-page-text-overrides-into-html.py --dry-run
  python catalog/merge-page-text-overrides-into-html.py
  git diff

Связанные файлы в репозитории: site-editor.js (подстановка на сайте),
catalog/api/read-page-text-overrides.php, catalog/api/save-page-text-overrides.php.
