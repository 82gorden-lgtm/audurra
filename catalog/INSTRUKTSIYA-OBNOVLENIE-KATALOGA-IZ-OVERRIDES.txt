================================================================================
  ОБНОВЛЕНИЕ КАТАЛОГА В РЕПОЗИТОРИИ: ВЛИТИЕ product-overrides.json
  Скрипт: catalog/merge-overrides-into-catalog.py
================================================================================

НАЗНАЧЕНИЕ
----------
На рабочем сайте редакторы могут менять карточки товаров через форму. Эти
правки PHP сохраняет в файле на сервере:

  catalog/api/data/product-overrides.json

Основной каталог в репозитории хранится в:

  catalog/catalog-products-data.js  (массив window.CATALOG_PRODUCTS)

Скрипт вливания переносит содержимое overrides в этот JS-файл по тем же
правилам, что и сайт (логика как в site-editor.js). После коммита обновлённого
catalog-products-data.js новый деплой покажет те же данные даже без файла
overrides на сервере.

ВАЖНО: скрипт НЕ изменяет и НЕ удаляет product-overrides.json. Он только
читает overrides и перезаписывает catalog-products-data.js (по умолчанию перед
этим создаётся резервная копия catalog-products-data.js.bak).


ЧТО НУЖНО НА КОМПЬЮТЕРЕ
----------------------
  - Установленный Python 3 (в терминале команда python или python3).
  - Клон репозитория New Site с нужной веткой.
  - Актуальный файл product-overrides.json с хостинга (см. ниже).


ОТКУДА ВЗЯТЬ product-overrides.json
-----------------------------------
Файл создаётся и обновляется на сервере при сохранении карточек в редакторе.
Способ копирования зависит от хостинга:

  - FTP / SFTP: скачайте файл по пути
    .../catalog/api/data/product-overrides.json
  - Панель хостинга (файловый менеджер): откройте тот же каталог и скачайте.
  - SSH: например
    scp user@host:/path/to/site/catalog/api/data/product-overrides.json .

Если на сервере правок ещё не было, файла может не быть — тогда вливать нечего.


КУДА ПОЛОЖИТЬ ФАЙЛ ЛОКАЛЬНО (УДОБНЫЙ ВАРИАНТ)
--------------------------------------------
Сохраните скачанный JSON в репозитории:

  catalog/api/data/product-overrides.json

Каталог data/ в проекте уже есть; сам JSON в .gitignore и в коммиты не попадает.

Либо сохраните файл в любое место и укажите путь флагом --overrides (см. ниже).


ПОРЯДОК РАБОТЫ (ПОШАГОВО)
-------------------------

1. Откройте терминал и перейдите в корень репозитория (папка New Site, где лежат
   index.html, каталог catalog/ и т.д.).

2. ПРОВЕРОЧНЫЙ ЗАПУСК без изменения каталога:

     python catalog/merge-overrides-into-catalog.py --dry-run

   На Linux или macOS при необходимости:

     python3 catalog/merge-overrides-into-catalog.py --dry-run

   В выводе смотрите:
   - сколько ключей (SKU) в overrides;
   - сколько из них реально влито в товары;
   - список SKU из overrides, которых НЕТ в catalog-products-data.js (такие
     записи пропускаются).

3. ОСНОВНОЙ ЗАПУСК (запись в catalog-products-data.js):

     python catalog/merge-overrides-into-catalog.py

   По умолчанию перед записью создаётся резервная копия:

     catalog/catalog-products-data.js.bak

   Файл .bak можно удалить после успешной проверки или оставить на всякий случай.
   Чтобы не создавать .bak, добавьте флаг --no-backup.

4. Откройте изменения в Git:

     git diff catalog/catalog-products-data.js

   Убедитесь, что правки соответствуют ожиданиям: названия, описания, ссылки,
   варианты, маркированные списки и т.д.

5. Закоммитьте и отправьте в удалённый репозиторий по вашему процессу (ветка,
   ревью, деплой на Beget и т.д.).


АЛЬТЕРНАТИВНЫЙ ПУТЬ К ФАЙЛУ OVERRIDES
-------------------------------------
Если JSON лежит не в catalog/api/data/product-overrides.json, укажите полный
путь к файлу, например:

  python catalog/merge-overrides-into-catalog.py --overrides "D:\Загрузки\product-overrides.json"

Редко нужно указать другой целевой JS:

  python catalog/merge-overrides-into-catalog.py --catalog-js путь\к\файлу.js


ФЛАГИ КОМАНДНОЙ СТРОКИ
---------------------
  --dry-run       Только статистика; catalog-products-data.js не меняется.
  --no-backup     Не создавать catalog-products-data.js.bak перед записью.
  --overrides     Путь к product-overrides.json, если не стандартный.
  --catalog-js    Путь к catalog-products-data.js, если не стандартный.


ПОСЛЕ ВЛИТИЯ: ФАЙЛ НА СЕРВЕРЕ
-----------------------------
Скрипт серверный product-overrides.json не трогает. Если данные уже в Git и
деплой подтянет обновлённый catalog-products-data.js, на хостинге по желанию
можно:

  - удалить из JSON записи по уже влитым SKU; или
  - заменить содержимое файла на пустой объект:  {}

Это необязательно: повторное применение тех же патчей обычно даёт тот же вид
карточки, но дубли в overrides со временем усложняют учёт правок.


ЧАСТЫЕ СИТУАЦИИ
---------------

  Файл overrides не найден локально
    Скрипт считает overrides пустыми; будет влито 0 позиций. Скачайте JSON
    с сервера или проверьте путь --overrides.

  В выводе перечислены SKU «нет в каталоге»
    В JSON есть SKU, которого нет в catalog-products-data.js. Такая запись
    пропускается. Нужно либо добавить товар в канон отдельно, либо убрать лишнюю
    запись из overrides.

  Ошибка при запуске Python
    Установите Python 3; в PATH должна быть команда python или python3.

  Очень большой diff
    Возможно при первом полном вливании. Скрипт пишет компактный JSON; важно
    проверить смысл изменённых полей, а не только размер diff.
