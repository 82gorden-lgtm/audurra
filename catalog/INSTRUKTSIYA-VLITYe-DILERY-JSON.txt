================================================================================
  ВЛИВАНИЕ dealers-ru.json В КАНОН РЕПОЗИТОРИЯ (карта «Где купить»)
  Скрипт: catalog/merge-dealers-json-into-repo.py
================================================================================

НАЗНАЧЕНИЕ
----------
На рабочем сайте редакторы после входа «Мой New Site» (право dealers) сохраняют
списки представителей по регионам. PHP записывает единый документ:

  catalog/api/data/dealers-ru.json

В репозитории этот же путь обычно держится как канон для деплоя (файл в Git).
Скрипт читает JSON (чаще всего скопированный с хостинга), приводит структуру к тому же
виду, что и на сервере (нормализация как в PHP cromax_normalize_dealers_document), и перезаписывает целевой файл с аккуратным форматированием.

Аналогия с другими зонами:
  - каталог    → catalog/merge-overrides-into-catalog.py + INSTRUKTSIYA-OBNOVLENIE-...
  - тексты     → catalog/merge-page-text-overrides-into-html.py + INSTRUKTSIYA-VLITYe-TEXT-BLOKOV-HTML.txt
  - дилеры     → этот скрипт + этот файл

ВАЖНО
-----
  - Скрипт не удаляет исходный файл; из него только читают.
  - По умолчанию и источник, и цель — catalog/api/data/dealers-ru.json в клоне
    репозитория (перезапись «на месте» после того, как вы подменили файл
    содержимым с сервера).
  - Если целевой файл уже есть, перед записью создаётся резервная копия *.bak,
    если не указан ключ --no-backup.


ЧТО НУЖНО НА КОМПЬЮТЕРЕ
----------------------
  - Установленный Python 3 (команда python или python3).
  - Клон репозитория New Site.
  - Файл dealers-ru.json с хостинга (если хотите не трогать локальный перед вливом).


ОТКУДА ВЗЯТЬ dealers-ru.json С СЕРВЕРА
--------------------------------------
Способ 1. Панель хостинга / FTP / SFTP: скачайте файл из каталога сайта:

    .../catalog/api/data/dealers-ru.json

Способ 2. Командная строка (пример с scp):

    scp user@host:/path/to/site/catalog/api/data/dealers-ru.json .

Сохраните куда удобно и укажите путь ключом --source при запуске скрипта.


КАК ЗАПУСТИТЬ
-------------
Из корня репозитория New Site.

1) Только статистика, без записи:

     python catalog/merge-dealers-json-into-repo.py --dry-run

   С источником «скачано с сервера»:

     python catalog/merge-dealers-json-into-repo.py --dry-run --source D:\downloads\dealers-ru.json

2) Запись в канон по умолчанию (catalog/api/data/dealers-ru.json):

     python catalog/merge-dealers-json-into-repo.py --source D:\downloads\dealers-ru.json

3) Если JSON уже подложен в локальный catalog/api/data/dealers-ru.json:

     python catalog/merge-dealers-json-into-repo.py


ПАРАМЕТРЫ
--------
  --source        Путь к входному dealers-ru.json (по умолчанию
                  catalog/api/data/dealers-ru.json в репозитории).

  --target        Куда записать результат (по умолчанию тот же путь).

  --dry-run       Не изменять файл, вывести version / число регионов / строк.

  --no-backup     Не создавать файл .bak у цели.


ЧТО СДЕЛАТЬ ПОСЛЕ ПРОГОНА
------------------------
  - Посмотреть изменения в Git: статус и diff именно файла dealers-ru.json.
  - Закоммитить при необходимости.
  - После следующего деплоя на сервере должен лежать обновлённый JSON там же в
    catalog/api/data/, если ваш процесс деплоя заливает файл из репозитория.
